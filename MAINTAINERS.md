## Submitting Updates

1. Open an issue or describe the change scope briefly.
2. Fork and create a feature branch.
3. Keep commits focused; follow existing code style and tooling.
4. Run relevant tests or manual checks (e.g., exercise key UI paths) and note results.
5. Open a pull request referencing the issue, including:
   - Summary of changes
   - Testing performed
   - Any risks/compatibility notes (AAP/AWX versions, gateway vs controller)
6. A maintainer will review; please respond to feedback promptly.
